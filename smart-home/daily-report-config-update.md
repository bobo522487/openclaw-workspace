## 🎯 HEARTBEAT 事件机制

**事件格式：** `[DAILY_MORNING_REPORT]`

**工作流程：**
1. Cron 任务在 08:15 (UTC 00:15) 触发
2. 发送消息 `[DAILY_MORNING_REPORT]` 给 agent
3. Agent 收到消息后，HEARTBEAT.md 中的逻辑生效
4. Agent 执行：
   - 获取贵阳天气
   - 搜索科技新闻（3条）
   - 搜索AI新闻（3条）
   - 生成并发送早报

## 🔧 技术细节

### Cron 表达式说明

**早报任务：** `15 0 * * *`
- 分钟：15
- 小时：0
- 日期：每天 *）
- 月份：每月 *）
- 星期：每天 *
- 含义：每天 UTC 00:15 执行
- 北京时间：UTC 00:15 + 8 = 08:15

**吃药提醒（保持不变）：**
- 早间：`0 1 * * *` (UTC 01:00 = 09:00)
- 午间：`0 7 * * *` (UTC 07:00 = 15:00)
- 晚间：`0 15 * * *` (UTC 15:00 = 23:00)

## 💡 注意事项

1. **HEARTBEAT.md 必须存在**
   - Agent 启动时会读取此文件
   - 如果文件不存在或格式错误，早报逻辑不会执行

2. **事件消息格式**
   - 必须是 `[DAILY_MORNING_REPORT]`（包含方括号）
   - 大小写敏感

3. **超时设置**
   - Agent 有 300 秒（5 分钟）的超时时间
   - 如果新闻搜索或天气获取超过此时间，任务可能会超时

4. **依赖检查**
   - 需要 `web_search` 工具可用（用于搜索新闻）
   - 需要 `weather` 或其他天气数据源
