## remindme vs OpenClaw cron: User Clarification Request
User asked: "remindme isn't just a skill?"

### Detailed Explanation Provided
**remindme = Natural Language Frontend + Cron Backend**

- remindme IS a skill (plugin) that sits on top of OpenClaw's cron system
- It parses natural language like "吃药 at 15:00" and calls OpenClaw's cron API
- Without remindme, you'd have to write cron expressions manually
- remindme makes it easier with natural language, but still depends on OpenClaw cron to actually execute

### Relationship
```
User → remindme (natural language)
       ↓
remindme parses time
       ↓
remindme calls OpenClaw cron API
       ↓
OpenClaw cron executes
```

### Why Distinguish?
| Aspect | remindme | OpenClaw cron |
|---------|----------|---------------|
| Position | Skill (plugin) | System feature (native) |
| Ease of use | ✅ Natural language | ❌ Requires cron expressions |
| Dependencies | ⚠️ Requires tsx | ✅ None |
| Scope | Reminders only | Any task |
| Flexibility | ✅ "吃药 at 12:14" | ❌ Manual config |

### Current Issues with remindme
- Missing dependencies: tsx, chrono-node, commander
- Installation complexity: Requires npm/pnpm
- Can't be used currently despite existing in workspace

### Decision: Use OpenClaw cron directly
- More stable
- No dependencies
- Already working (confirmed by tests)
- Luna can help configure quickly
