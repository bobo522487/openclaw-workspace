### 🌟 MCP迁移
**时间：** 2026-02-09 11:00 UTC

**问题：** 直接使用Home Assistant REST API需要手动管理entity_id和构造API请求，开发效率低

**解决方案：** 配置ha-mcp MCP服务器，提供80+标准化工具

**配置文件：** `~/.openclaw/openclaw.json`
```json5
{
  "agents.defaults.mcp.servers": [
    {
      "name": "homeassistant",
      "command": "npx",
      "args": ["-y", "@homeassistant-ai/ha-mcp"],
      "env": {
        "HA_URL": "http://10.10.10.8:8123",
        "HA_TOKEN": "eyJhbGc..."
      }
    }
  ]
}
```

**遇到的问题：**
- openclaw.json权限问题（root:root）
- Gateway无法读取配置

**解决：**
- 使用Python脚本修改配置文件，保持正确权限
- 重新配置并重启Gateway

**优势：**
- ✅ 无需手动管理entity_id
- ✅ 80+预置工具 vs 之前的1-2个API调用
- ✅ 智能设备发现
- ✅ 语义化接口，AI更易理解
- ✅ 支持自动化、日历、待办等高级功能
